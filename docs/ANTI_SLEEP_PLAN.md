# План: как не дать сайту засыпать

## Текущая ситуация

- **Платформа:** Render free tier
- **Засыпание:** после **15 минут** без входящих запросов
- **Пробуждение:** cold start ~**1 минута** (иногда 25–60 сек)
- **Проблема:** пользователь ждёт, не понимая, что происходит

---

## Решения (от простого к сложному)

---

## Вариант 1: Оставить Render + внешний пинг (рекомендуется)

**Идея:** раз в 5–10 минут внешний сервис дергает `/api/ping` — Render считает, что есть трафик, и не укладывает сервис спать.

### Бесплатные сервисы для пинга

| Сервис | Интервал | Ограничения |
|--------|----------|-------------|
| **UptimeRobot** | 5 мин | 50 мониторов, бесплатно |
| **cron-job.org** | от 1 мин | Без лимитов по заданиям |
| **GitHub Actions** | мин. 5 мин | Бесплатно для публичных репо |

**Минимальный интервал для Render:** 10 минут (если пинг каждые 10 мин, за 15 мин будет 2 запроса — сервер не уснёт).

### Пошагово (UptimeRobot)

1. Регистрация на https://uptimerobot.com
2. **Add New Monitor**
3. Настройки:
   - Monitor Type: **HTTP(s)**
   - Friendly Name: `Checker FB ping`
   - URL: `https://checker-account-fb.onrender.com/api/ping`
   - Monitoring Interval: **5 minutes**
4. Create Monitor

**Результат:** сервер бодрствует, cold start почти не будет.

---

## Вариант 2: GitHub Actions как пинг (всё в репо)

Без внешних сервисов, всё в GitHub.

1. Создать `.github/workflows/keep-alive.yml`:

```yaml
name: Keep Render Awake
on:
  schedule:
    - cron: '*/10 * * * *'   # каждые 10 минут
  workflow_dispatch:         # ручной запуск
jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: Ping Render
        run: |
          curl -s -o /dev/null -w "%{http_code}" "https://checker-account-fb.onrender.com/api/ping"
```

2. Файл коммитится в репо — workflow начнёт выполняться по расписанию.

**Результат:** бесплатно, внутри проекта, без сторонних аккаунтов. Интервал минимум 5 минут (`*/5 * * * *`).

---

## Вариант 3: Переехать на Cloudflare Workers (без сна)

**Идея:** прокси-логика выносится в Cloudflare Worker. Workers на edge, **без засыпания**, почти нет cold start.

- 100 000 запросов/день бесплатно
- Ответ за миллисекунды
- Статика остаётся на GitHub Pages
- `config.js` — `PROXY_BASE` указывает на Worker URL

**Минусы:**
- Нужна переработка бэкенда (Express → Worker)
- Отдельный деплой на Cloudflare

---

## Вариант 4: Cloudflare Pages + Functions

Аналог Workers, но через Pages:
- Статика — Cloudflare Pages
- `/api/get_uid/:id` — Pages Function (тот же прокси)
- Без засыпания, бесплатный лимит щедрый

**Минусы:** более заметная переработка (деплой, структура).

---

## Вариант 5: Платформы с такими же ограничениями

| Платформа | Sleep | Cold start | Примечание |
|-----------|-------|------------|------------|
| **Render** | 15 мин | ~1 мин | Текущий |
| **Heroku** | 30 мин | ~30 сек | Свободного tier почти нет |
| **Koyeb** | да | 1–5 сек | Похоже на Render |
| **Railway** | нет | — | Pay-per-use, после бесплатных $5 платно |
