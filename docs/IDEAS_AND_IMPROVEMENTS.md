# Идеи для нового функционала и улучшений

Анализ текущего чекера и предложения по развитию.

---

## Новый функционал

### 1. Загрузка файла (drag & drop или выбор)

**Суть:** загружать список из `.txt` / `.csv` вместо вставки в textarea.

**Полезность:** удобно для больших списков (1000+ строк). Меньше риск зависания при вставке в браузер.

**Сложность:** низкая. `input type="file"` + `FileReader.readAsText()`.

---

### 2. Экспорт в один файл с метками

**Суть:** кнопка «Сохранить всё» — один файл вида:
```
[VALID] 100012345678901
[VALID] 100012345678902
[BLOCKED] 100012345678903
```

**Полезность:** один файл для дальнейшей обработки (скрипты, фильтры).

**Сложность:** низкая.

---

### 3. Фильтрация/поиск в результатах

**Суть:** поиск по тексту внутри полей «Валидные» и «Невалидные».

**Полезность:** если результатов много, быстро найти нужный ID.

**Сложность:** средняя (добавить поле поиска + фильтрацию строк).

---

### 4. История проверок (localStorage)

**Суть:** хранить последние N проверок (дата, количество, валид/невалид) и показывать в блоке «История».

**Полезность:** сравнение результатов между запусками.

**Сложность:** средняя. Нужна структура хранения и UI.

---

### 5. Настройка параллелизма

**Суть:** слайдер или поле «Кол-во одновременных запросов» (10–50).

**Полезность:** при риске rate limit — снизить; при быстром интернете — повысить скорость.

**Сложность:** низкая. Вынести `CONCURRENCY` в настройки.

---

### 6. Повторная проверка только невалидных

**Суть:** кнопка «Проверить заново невалидные» — берёт список из блока «Невалидные» и перепроверяет.

**Полезность:** снижение числа ложных срабатываний из‑за временных сбоев.

**Сложность:** низкая.

---

### 7. Поддержка других форматов строк

**Суть:** извлекать ID из ссылок `facebook.com/profile.php?id=XXX`, `m.facebook.com/XXX` и т.п.

**Полезность:** можно вставлять ссылки, а не только чистые ID.

**Сложность:** средняя. Расширить regex / парсер.

---

### 8. Оценка времени до конца проверки

**Суть:** по скорости первых N запросов оценивать оставшееся время.

**Полезность:** понятно, сколько ещё ждать.

**Сложность:** средняя.

---

## Улучшения существующего

### 9. Доступность (a11y)

- Корректные `aria-label` у кнопок.
- Фокус-менеджмент: при появлении результатов — фокус на «Валидные», Tab по порядку.
- Сообщения для screen readers при обновлении прогресса.

**Сложность:** низкая–средняя.

---

### 10. Оффлайн / ошибка сети

- Более явное отображение ошибок (красный блок, иконка).
- Разделение «сеть недоступна» и «сервер не отвечает».
- Подсказка, что делать (проверить интернет, подождать, обновить).

**Сложность:** низкая.

---

### 11. Состояние stopBtn

- Сейчас `stopBtn.disabled = true` после нажатия, но не сбрасывается после завершения.
- Нужно в `finally` восстановить `stopBtn.disabled = false`.

**Сложность:** минимальная.

---

### 12. Debounce для updateInputStats

- При вставке 1000 строк счётчик обновляется при каждом `input`.
- Добавить debounce 200–300 ms, чтобы снизить нагрузку.

**Сложность:** минимальная.

---

### 13. Подсказка по формату ID

- Плейсхолдер или tooltip: какие ID поддерживаются (10/61 + 10–23 символа).

**Сложность:** минимальная.

---

### 14. Сохранение результатов при закрытии вкладки

- Перед `beforeunload` сохранять данные в `sessionStorage`, при возврате — предлагать восстановить.

**Полезность:** при случайном закрытии вкладки не терять результат.

**Сложность:** средняя.

---

### 15. Тёмная тема

- Сейчас только светлая. Добавить переключатель темы и CSS‑переменные для тёмного режима.

**Сложность:** средняя.

---

### 16. Мобильная вёрстка

- Проверить и подправить на маленьких экранах: кнопки, текстовые поля, прогресс.

**Сложность:** низкая–средняя.
